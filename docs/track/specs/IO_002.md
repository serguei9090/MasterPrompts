# Technical Specification: IO_002 - Path Variable Resolution

## 1. Executive Summary
This task implements standardized path variable resolution in the core agent prompts and configuration files. This ensures that the framework is truly portable and can resolve absolute paths based on the project's root directory, even when running in different environments or shells.

## 2. Problem Statement
- **Context**: Hardcoded paths (e.g., `I:\01-Master_Code\...`) in prompts and rules make the framework brittle. The AI needs a way to resolve paths like `{{project_root}}` or `@/docs` dynamically.
- **Goals**:
    - Define a standardized set of path variables.
    - Implement a "HAL" (Hardware Abstraction Layer) script or directive that resolves these variables before the agent executes a command.

## 3. Technical Architecture
- **Layer Impact**: System Rules (`.agents/rules/System/HAL_Interoperability.md`), Environment Configuration (`agents.yaml`), and potentially a pre-execution script.
- **Data Model**: Update `agents.yaml` (if it exists) to include path mappings.
- **API Definition**: Variable syntax: `{{VAR_NAME}}`.

## 4. Implementation Plan
1. **Define Variables**: Identify core paths: `ROOT`, `AGENTS`, `DOCS`, `TRACK`.
2. **Update HAL Rule**: Create or update `.agents/rules/System/HAL_Interoperability.md` to define the resolution logic.
3. **Environment Sync**: Create a `scripts/hal-sync.ps1` that the agent can run to export these variables to the current session's environment.
4. **Integration**: Update `AGENTS.md` and key workflows to use the new variables instead of hardcoded paths.

## 5. Verification Plan
- **Test Case 1**: Run a command using a path variable (e.g., `ls {{TRACK}}`) and verify it resolves to the correct local directory.
- **Test Case 2**: Verify the framework can be "migrated" to a temporary directory and still resolve paths correctly.

## 6. Risk Assessment
- **Variable Collisions**: Ensure variables don't conflict with system environment variables.
- **Shell Compatibility**: The resolution must work across PowerShell (local) and potentially Bash (remote/Docker).
