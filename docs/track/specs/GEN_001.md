# Technical Specification: GEN_001 - Diagram-Creator Integration

## 1. Executive Summary
This task involves the functional "linking" of the `diagram-creator` skill to the `/auto-improve` workflow cycle. The goal is to ensure that any architectural refinement or rule pruning that alters the framework's structure results in an automatically generated or updated visual map following the AVAS standard.

## 2. Problem Statement
- **Context**: The `/auto-improve` workflow currently mentions visualization as a step, but it lacks specific triggers and procedural hooks to ensure the `diagram-creator` skill is used effectively.
- **Goals**: 
    - Automate the trigger for `diagram-creator` when a structural change is detected.
    - Ensure updated diagrams are persisted in `docs/architecture/` and linked in `docs/track/TODO.md` or `telemetry.csv`.

## 3. Technical Architecture
- **Layer Impact**: Operational Layer (`.agents/workflows/Auto/AutoImprove.md`), Capability Layer (`.agents/skills/diagram-creator/`).
- **Data Model**: No schema changes required.
- **API Definition**: Procedural "hook" within the markdown workflow.

## 4. Implementation Plan
1. **Audit Workflow**: Review `.agents/workflows/Auto/AutoImprove.md` to identify the exact injection point.
2. **Procedural Update**: Modify the "Step 4: Verify, Apply & Visualize" section to be more explicit about triggering the skill.
3. **Mnemonic Anchor**: Add a directive to the `diagram-creator` skill to acknowledge it is part of the `auto-improve` cycle.
4. **Verification**: Simulate a rule update and verify the agent's attempt to trigger the skill.

## 5. Verification Plan
- **Test Case 1**: Run a mock `/auto-improve` session where a directory is added/renamed. Verify the agent calls `activate_skill("diagram-creator")`.
- **Test Case 2**: Ensure the generated diagram is saved to `docs/architecture/diagrams.md` (or a dedicated file).

## 6. Risk Assessment
- **Token Usage**: Automatic diagram generation can be token-intensive if triggered too frequently. The hook must be conditional on *structural* changes only.
