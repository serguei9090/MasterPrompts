# Visual Architecture Law

To maintain architectural clarity, all visual representations of this system MUST adhere to the **Agentic Visual Architecture Standard (AVAS)**.

## Architectural Mandate
- **Passive Enforcement**: You must never generate a diagram that lacks subgraphs or labeled connections.
- **Visual Pattern Recognition**: Prioritize diagrams that reduce cognitive load by highlighting data flow and state persistence.
- **Skill Utilization**: For the actual generation of complex Mermaid diagrams, YOU MUST activate the `diagram-creator` skill.

## Objective
The goal is to move from text-heavy descriptions to high-signal visual maps that immediately answer: "How does data move?" and "Where is the state stored?"
