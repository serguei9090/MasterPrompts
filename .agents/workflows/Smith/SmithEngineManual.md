# Smith Engine (Manual)

## Global Objective
Execute the software development cycle with explicit user gating at critical decision points.

### Phase 0: Research & Propose
**Assume Role:** `@lead`
1. **Intelligence Sync**: Run `codanna` impact analysis and `cognee` memory recall.
2. **Propose**: Present a multi-step plan via `sequentialthinking`.
3. **GATE**: `ask_user` for approval of the plan and `bd` tasks.

### Phase 1: Design Spec
**Assume Role:** `@ui-designer`
1. **Tokens**: Propose updates to `DESIGN.md` tokens and Atomic hierarchy.
2. **GATE**: `ask_user` for approval of the visual spec before implementation.

### Phase 2: Implementation
**Assume Role:** `@builder`
1. **Surgical Coding**: Build the logic/UI based on approved specs.
2. **Hygiene**: Enforce `DESIGN.md` tokens and the `callSidecar` bridge protocol.

### Phase 3: Audit & Review
**Assume Role:** `@auditor`
1. **Verify**: Run tests, linters, and visual parity audit.
2. **GATE**: Present the audit report and `ask_user` for approval to proceed to release.

### Phase 4: Distillation & Sync
**Assume Role:** `@lead`
1. **Distill**: Run `cognee remember` for lessons learned.
2. **Sync**: Update `bd` status, conventional commit, and `git push`.
